## What We Learned
- A short five-episode run is enough to validate the pipeline, but not enough to claim a strong trading policy.
- Comparing DQN return with Buy-and-Hold prevents us from over-celebrating raw portfolio growth.
- Trade count is important because reward bugs or missing costs can create over-trading.
- Sharpe and drawdown expose instability that total return alone can hide.

## Struggles and Fixes During Development
- A corrupted parquet cache initially broke AAPL preparation; `YFinanceDataClient` now ignores corrupt cache and retries the normal path.
- yfinance's internal SQLite cache failed on this machine until the client routed it into `data/raw/.yfinance-cache`.
- The first README visuals were generated previews and did not fully match the live GUI; the real Tkinter styling was updated afterward.
- The project deliberately avoids fake results. Metrics in this report come from the local experiment runner.
- SPY/NVDA comparison should be rerun on a machine where yfinance TLS verification succeeds, or by providing CSV fallback files.
